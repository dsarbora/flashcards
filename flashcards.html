JavaScript

A powerful programming language that makes websites like this one interactive.

Operators

Operators include + (addition), - (subtraction), * (multiplication), / (division, and % (modulo, taking the percentage of), as a number of logical operators (||, &&)

Variables

Variables are a powerful programming tool. They are used when a piece of date may not be the same in every interation. userName would be considered a variable.

Variable Naming Conventions

Variable names should be explanatory but brief. If you name a variable "a," you may forget it or another developer will have to figure out what it's for. Variables usually begin with a lower-case letter, and any subsequent words are marked with a capitalFirstLetter without spaces.

Functions

Functions do something in JavaScript. You can use a function to do almost anything, which can be a source of endless joy or frustration when you accidentally tell a function to do something you don't want it to do, or forget to tell it do do something you want it to do.

Methods

Methods are things that you can call in a couple of words that would otherwise be much more complicated to write as functions. You can use them to perform simple, useful tasks like returning the length of a string, or splitting that same string into its component characters and pushing those characters into an array.

Arguments

Arguments are the real data that your function uses. They are variables representing integers, strings, characters, or booleans that functions use to do a thing. Arguments are actual data.

Parameters

Parameters are hypothetical data that your function is meant to use to get something done. The number of parameters is how many you should use when calling it, and the names of the parameters will remind you or another developer what type of data they should be. It is important that a function's internal variables match the parameters, not that any external variables do.

Return

In order to get a usable answer to a math problem from a computer you have to tell it to take your two numbers, return with the answer to your specific problem, and set something equal to the value they returned with.

Chaining Methods

There is no limit to loops inside of loops, functions calling functions, or being inside of other functions, or the number of methods you can place one after the other. If the logic is correct, the program will do as it's told. If the logic is incorrect, the program will unfortunately also do exactly as it's told.

Strings

A string is a data type consisting of characters that isn't interpreted as a number. It can be made of what look to you and me like numbers, but a computer will not treat it as such unless told to do so. Data types made of numbers may be integers or strings, so keep them straight!

Booleans

Booleans are true or false values. "They're great!" = true

Undefined

"Undefined" is what you see when you try to ask a computer for the value of a variable, but you haven't yet told the computer what exactly the variable is. Asking a computer to divide 5 by 7 and asking it for the answer isn't going to be enough. You have to tell a computer that the answer is 5 divided by 7, and then you can ask what the answer is.

NaN

Not a Number... this data type often means you've made a mistake in taking a string and trying to divide it by a number or vice versa, at least in the early stages.

Escape

Sometimes you need a good escape. Seems to be a way for a computer to read umlauts and accents for letters in other languages.

Alert

Alerts cause your browser to make a popup window with a message. They are used to alert the user to something in particular, pretty much guaranteeing that they will catch one's attention.

Comments

Comments in JavaScript are single line // or multi-line /* */. They can be useful for leaving messages to yourself or another coder, or to temporarily try a program without a particular piece of code.

jQuery

jQuery is a JavaScript library. It contains methods that can perform a number of actions.

Attributes

Attributes are added to elements as a way of identifying them in your code. Right click on this text to see that one of the attributes associated to it is the class "attributes."
